# 모바일 개발자 미니 해커톤 과제 안내

안녕하세요.  
이번 미니 해커톤에서는 제공된 요구사항을 기반으로 **모바일 앱 프로토타입**을 구현합니다.

이 과제의 목적은 단순한 화면 클론이 아니라, 제한된 시간 안에서 **디자인 해석력, 제품 기획력, 모바일 구현 역량, 아키텍처 의사결정 능력**을 함께 확인하는 것입니다.

- Figma: <https://www.figma.com/design/XqqPeEoWsD5tBmil5Z7a50/mini_hack?node-id=0-1&p=f&m=dev>

---

## 1. 과제 주제

제공된 Figma는 사용자가 자신의 경험, 장소, 취향, 일상을 기록하고 이를 **캘린더, 라이브러리, 기록 카드** 형태로 관리하는 모바일 앱 디자인입니다.

참가자는 이 디자인을 바탕으로 다음 기능을 가진 앱을 구현합니다.

> 사용자가 자신의 기록을 만들고, 날짜별로 확인하며, 기록 카드를 생성/편집할 수 있는 모바일 앱

백엔드 서버는 사용하지 않으며, 앱 내부 데이터만으로 동작해야 합니다.

---

## 2. 구현 범위
구현은 최대한 모든 범위를 구현하는 걸로 목표하되, 디자인 싱크가 높아야 합니다.
하단의 기능 명세는 실제 피그마의 이미지와 상이할 수 있으니, 검증이 필요합니다.

### A. 라이브러리 / 캘린더 화면

Figma의 `라이브러리` 화면을 참고하여 구현합니다.

기능 명세:

- 월 단위 캘린더 UI
- 날짜별 기록 표시
- 기록이 있는 날짜에는 이미지, 아이콘, 배지 등으로 시각적 표시
- 상단 탭 영역 표현
  - 예: 위시리스트, 캘린더, 평점, 타임라인, 갤러리, 장르
- 필터 칩 UI 표현
  - 예: 평점, 날짜, 전체, 카테고리
- 하단 탭(단, 각 플랫폼에 맞게 생성 가능. flutter는 양 플랫폼 선택 혹은 커스터마이징) / 중앙 생성 버튼 구현
- 데이터는 서버 없이 앱 내부 샘플 데이터 또는 로컬 저장소로 구성

---

### B. 기록 생성 플로우

사용자가 새로운 기록을 추가하는 흐름을 구현합니다.

기능 명세:

- 기록 생성 진입
- 제목, 날짜, 카테고리, 이미지 또는 대표 아이콘 선택
- 저장 시 캘린더 화면에 반영
- 최소 1개 이상의 입력값 검증
  - 예: 제목 미입력 시 저장 불가
- 생성 완료 화면 구현
  - 예: `첫 번째 기록 완료!`
  - `기록 카드 만들기`, `다른 기록하기`, `홈` 액션 제공

---

### C. 기록 카드 화면

Figma의 기록 카드 / 뷰어 화면을 참고하여 구현합니다.

기능 명세:

- 생성된 기록을 카드 형태로 표시
- 카드에는 최소 다음 정보 포함
  - 제목
  - 날짜
  - 카테고리 또는 설명
  - 대표 이미지 / 배경
- 카드 미리보기 화면
- 배경 편집 또는 스타일 변경 기능 중 1개 이상 구현
  - 예: 배경 색상 변경
  - 예: 카드 프레임 변경
  - 예: 텍스트 위치 또는 테마 변경
- 저장, 공유, 스토리 버튼은 실제 외부 연동 없이 UI 또는 로컬 동작으로 처리 가능

## 3. 디자인 요구사항

Figma에 포함된 디자인을 편하게 사용해도 되고, 복사해서 다른 도구를 사용해도 됩니다.
단, 디자인 싱크가 높아야 합니다.

## 4. 기술 요구사항

### 아키텍처 / 디자인 패턴

하나 이상의 디자인 패턴 또는 아키텍처를 반드시 적용해야 하며, 왜 해당 패턴/아키텍처를 적용했는지 문서화하세요.
그리고 해당 패턴/아키텍처에 대한 고민을 세션 로그에 드러나게 하세요.

예시:

- MVVM
- MVI
- Clean Architecture
- Repository Pattern
- Redux / Unidirectional Data Flow
- Coordinator / Router Pattern
- State Machine
- UseCase Layer
